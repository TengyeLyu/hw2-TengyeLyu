## Step 2: Choose a business workflow and define the task

### Workflow chosen: 
Drafting customer support email responses.

### User:  
A customer support agent at an online retail company.

### System input: 
The system receives a customer message or complaint, such as a delivery issue, refund request, or account problem.

### System output:  
The system produces a polite and professional draft email reply for the support agent to review or send.

### Why this task is valuable to automate or partially automate: 
This task is repeated often and takes time. A draft response can help agents reply faster, keep tone consistent, and reduce routine writing work. It is a good task for partial automation because low-risk cases may be handled quickly, while more sensitive cases can still be reviewed by a human.
